### File Storage Service

**Назначение**: хранение загружаемых файлов в локальной файловой системе и выдача их по идентификатору.

### Эндпоинты

- **GET** `/healthz`  
  **Описание**: health-check сервиса.  
  **Ответ 200**:
  ```json
  { "status": "ok", "service": "file-storage" }
  ```

- **POST** `/files`  
  **Описание**: загрузка файла.  
  **Тело запроса**: `multipart/form-data` с полем `file`.  
  **Ответ 201**:
  ```json
  { "file_id": "string" }
  ```
  **Коды ошибок**:
  - `400` — нет файла или некорректная форма.
  - `413` — файл слишком большой.
  - `500` — ошибка сохранения.

- **GET** `/files/{id}`  
  **Описание**: получение файла по идентификатору.  
  **Ответ 200**: бинарное содержимое файла, заголовок `Content-Disposition` с оригинальным именем.  
  **Коды ошибок**:
  - `400` — не указан `id`.
  - `404` — файл не найден.
  - `500` — ошибка чтения.

### Примеры curl

- **Health-check**:

```bash
curl -i http://localhost:8081/healthz
```

- **Загрузка файла**:

```bash
curl -i -X POST http://localhost:8081/files \
  -F "file=@/path/to/work.txt"
```

- **Получение файла**:

```bash
curl -i http://localhost:8081/files/{file_id}
```
